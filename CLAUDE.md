# GrantMatch API — Project Context

## What this is
A production ML REST API that accepts a project/company profile and returns
a ranked list of matching UK and EU grants with fit scores, eligibility
signals, and deadlines. No competitor does ML-scored grant matching —
existing tools are keyword search only.

## Architecture
- FastAPI handles HTTP, Pydantic v2 validates I/O
- Grant data ingested from: UKRI GtR API, CORDIS bulk JSON, GOV.UK
  Find a Grant scrape, Innovate UK UKRI opportunities page scrape
- Grants stored in SQLite (dev) / PostgreSQL (prod) with full-text +
  vector search
- sentence-transformers (all-MiniLM-L6-v2) embeds grant descriptions
  and applicant profiles
- FAISS vector index enables semantic retrieval of top-K candidates
- XGBoost reranker trained on grant-profile pairs scores final ranking
- SHAP explains top-3 factors per matched grant
- Celery + Redis (optional) for async grant data refresh jobs

## Key files
- app/main.py — FastAPI app entry point, startup events
- app/api/routes.py — all endpoints
- app/models/schemas.py — Pydantic request/response models
- app/services/embedder.py — sentence-transformer singleton
- app/services/matcher.py — core matching pipeline
- app/services/reranker.py — XGBoost reranker, loads model or uses heuristic
- app/utils/eligibility.py — rule-based eligibility checks per grant type
- app/utils/feature_extractor.py — builds feature vector for reranker
- data/ingest/ — ingestion scripts for each data source
- data/grants.db — SQLite grant database (gitignored, generated by ingest)
- ml/train.py — trains XGBoost reranker on synthetic pairs
- ml/model.pkl — trained model (gitignored)
- tests/ — pytest suite

## Environment variables (.env)
- DATABASE_URL=sqlite:///data/grants.db
- EMBEDDING_MODEL=all-MiniLM-L6-v2
- MODEL_PATH=ml/model.pkl
- FAISS_INDEX_PATH=data/grants.faiss
- API_KEY=changeme
- MAX_RESULTS=20
- GtR_API_BASE=https://gtr.ukri.org/gtr/api
- UKRI_OPPORTUNITIES_URL=https://www.ukri.org/opportunity/

## Data sources
1. UKRI GtR API — funded project data (historical signal for what gets funded)
   Base: https://gtr.ukri.org/gtr/api
   Format: JSON, paginated, no auth required
2. CORDIS Horizon Europe bulk data — EU grant project data
   Download: https://data.europa.eu/data/datasets (he_project CSV/JSON)
   Format: CSV/JSON bulk download, open license
3. GOV.UK Find a Grant — UK government grants
   URL: https://www.find-government-grants.service.gov.uk/grants
   Format: HTML scrape (no public API — use requests + BeautifulSoup)
4. UKRI Opportunities — live Innovate UK competitions
   URL: https://www.ukri.org/opportunity/
   Format: HTML scrape

## Do not
- Do not use any paid data sources or require API keys for data ingestion
- Do not load embedding model on every request — cache at startup
- Do not use OpenAI or any external LLM
- Do not store raw HTML — parse and store structured grant records only
- Keep SQLite for dev, design schema so PostgreSQL swap is one env var change
